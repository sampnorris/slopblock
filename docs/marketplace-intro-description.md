# Introductory Description

<!-- The opening paragraph of the Marketplace listing. Sets context before the feature list. -->

SlopBlock blocks a pull request until the author passes a short quiz based on the actual diff. When a PR opens, it reads the changed code, writes multiple-choice questions about behavior and risk, checks those questions with a second model, and posts the quiz link back to the PR. Docs-only changes, tests, lockfile bumps, formatting, and other low-signal updates are skipped automatically. The point is simple: if you're going to merge the code, you should be able to explain it.
